# 飞书数字指挥中枢 (v1.0.3)

> 效率决定结果，分寸决定未来。

本技能通过双轨运行协议，解决飞书深度集成中的安全与授权边界问题。
同时在执行前加入一层**协作诊断内核**：先判断问题属于消息、审批、会议、表格、日程还是沟通分寸，再决定该摘要、草拟、提醒、协调还是暂缓。

## 核心特性

- **初始化握手**：首次调用时明确运行级别（参谋模式 / 执行模式）
- **默认克制**：未显式授权前，默认处于只读与草稿模式
- **高敏红线**：向上汇报、跨部门公开催办、审批终审、核心表格修改等动作始终保留二次确认
- **工程透明**：Instruction-only 设计，无内置网络代码，依赖宿主平台 Connector

## 运行模式

### 1. 参谋模式（默认）
AI 负责读取、分析、总结、草拟。  
所有写操作都必须由用户显式确认。

### 2. 执行模式
AI 可在授权范围内处理常规协作动作。  
但高敏动作仍然必须二次确认。

## 工程身份（Engineering Identity）

- **性质：** Instruction-only Orchestrator
- **依赖：** 底层连接依赖宿主平台的 Feishu Connector 完成
- **凭证：** 技能本身不持久化任何飞书凭证
- **原则：** 遵循最小动作原则与私有上下文原则

## 适用环境

- 需要处理海量飞书消息、审批、会议与表格协作的中层管理者
- 对协作效率要求极高、同时对权限边界极其敏感的企业环境
- 希望把飞书从“消息入口”升级为“行动中枢”的团队
